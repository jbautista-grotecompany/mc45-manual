    6 Deposit Sequence & Process Control¶

    6.1 Deposit Cycle Overview¶

    The MC-45 deposits topping in a repeating cycle triggered by each target detected by the TARGET SENSOR. The cycle runs as follows:

    1. The PRODUCT CONVEYOR runs continuously. The TARGET SENSOR detects a target on the PRODUCT CONVEYOR.
    2. The TARGET DELAY elapses. The CARPET trigger fires.
    3. The CARPET extends from the MASK OFFSET position. It travels in sync with the target at a speed proportional to the PRODUCT CONVEYOR speed, set by the EXTEND RATIO recipe parameter. As the CARPET extends over the target, it folds mechanically to conform to the target surface, positioning the deposit zone over the border area.
    4. The CARPET retracts to the MASK OFFSET position.
    5. The PORTION CONVEYOR advances the length set by PORTION CONV MOVE LENGTH, loading the next portion of topping toward the BUCKET.
    6. The BUCKET FALL TIME elapses, allowing topping to fall from the PORTION CONVEYOR and reach the CARPET surface before the BUCKET opens.
    7. The BUCKET opens for the duration of BUCKET OPEN TIME. Topping falls through the MASK onto the CARPET. The SPINNER runs continuously and spreads the topping across the CARPET as it lands.
    8. The BUCKET closes. The CARPET is now loaded and ready for the next target.
    9. The cycle repeats on the next TARGET SENSOR detection.

    In SKIP MODE, the CARPET trigger, BUCKET, and PORTION CONVEYOR all skip every other target. On a skipped target, the CARPET stays at the MASK OFFSET position, the BUCKET stays closed, and the PORTION CONVEYOR does not advance.

    If the skip phase drifts out of alignment with the line — for example, when an operator removes a target to check deposit quality — press SYNC on the OI. Each press shifts the skip phase by one target, realigning the deposit pattern with the correct targets.

    6.2 CARPET Motion¶

    The CARPET is a retractable belt that positions the target under the deposit zone. It does not return to the home position between cycles. Between cycles it holds at the MASK OFFSET position, which is an offset from home set in the active recipe.

    MASK OFFSET sets where the CARPET starts its extend move, effectively aligning the deposit zone with the border of the target.

    STROKE LENGTH controls the width of the deposit zone.

    EXTEND RATIO controls how fast the CARPET travels relative to the PRODUCT CONVEYOR during the extend move.

    The MASK is mounted between the BUCKET and the CARPET surface. It shapes the topping pattern — round masks produce round deposits, square cells produce square deposits — so the topping lands only within the border area defined by MASK OFFSET and STROKE LENGTH.

    6.3 Target Tracking¶

    The Depositor tracks each target from the moment the TARGET SENSOR detects it through the deposit cycle. Three tracking modes are available, selected from the service screen on the OI.

    Time-Based Tracking¶

    The PLC uses a five-timer memory chain to track the target's position after detection. Each timer represents a fixed time segment of the target's travel from the TARGET SENSOR to the CARPET trigger point. The delay between detection and trigger is set by TARGET DELAY in the active recipe.

    Operators can fine-tune the trigger point from the HOME screen using the FINE and COARSE adjustment buttons, which update TARGET DELAY in the active recipe directly.

    This mode requires no encoder. It is the standard production mode for most installations.

    Encoder-Based Tracking¶

    When a PRODUCT CONVEYOR encoder is installed, the PLC tracks target position by distance rather than time. The encoder on the PRODUCT CONVEYOR shaft fires the CARPET trigger when the target has traveled the distance set by TARGET DELAY.

    This mode compensates for line speed variations automatically. It is the preferred mode when conveyor speed changes frequently during production.

    Test Mode¶

    Test Mode fires the CARPET trigger on a repeating timer interval set by TARGET MODE PRESET on the service screen. No TARGET SENSOR input is required. The CARPET cycles continuously at the set interval regardless of whether targets are present on the PRODUCT CONVEYOR.

    Test Mode is used during commissioning and mechanical setup to verify CARPET motion and deposit placement without running product.

    6.4 HOPPER Fill Control¶

    The RAKE maintains an even topping bed on the PORTION CONVEYOR. The RAKE runs continuously at the speed set by RAKE SPEED in the active recipe, leveling the bed to a consistent depth. As the BUCKET removes topping each cycle, the bed level drops. The HOPPER fill system replenishes topping automatically.

    Two load cells monitor topping weight: one on the PORTION CONVEYOR (HOPPER) and one on the FEED CONVEYOR (when equipped). The HOPPER LOW WEIGHT and HOPPER HIGH WEIGHT recipe parameters set the start and stop thresholds for the FEED-Z CONVEYOR.

    When HOPPER weight drops below HOPPER LOW WEIGHT, the FEED-Z CONVEYOR starts and delivers topping to the HOPPER area. When HOPPER weight reaches HOPPER HIGH WEIGHT, the FEED-Z CONVEYOR stops.

    6.5 FEED-Z CONVEYOR — Two-Machine Configuration (Option)¶

    When two MC-45 Depositors share one FEED-Z CONVEYOR, the conveyor is bidirectional. Running forward feeds MC45 Machine 1. Running in reverse feeds MC45 Machine 2.

    Machine 1 takes priority. When both machines demand topping simultaneously, the FEED-Z CONVEYOR runs forward to serve Machine 1 first. Machine 2 waits until Machine 1 is satisfied or until the Machine 2 timeout expires. The timeout value is set from the OI.

    The FEED CONVEYOR has its own load cell. The FEED CONVEYOR weight thresholds in the active recipe control FEED-Z CONVEYOR start and stop independently of the HOPPER load cell.

    6.6 Supply and Demand Balance¶

    The FEED-Z CONVEYOR cycles on and off to maintain topping weight between HOPPER LOW WEIGHT and HOPPER HIGH WEIGHT. The balance between topping consumption and replenishment rate determines how frequently the FEED-Z CONVEYOR runs and how stable the HOPPER weight stays during production.
